FloorAI / Slide 2

The product is more than chat: it is an evented operations workspace with evaluation built in.

Architecture flow

Step 1
Convex as canonical state
Issues, policies, staffing, files, action items, messages, message events, answer packets, eval runs, and audit logs live in one backend.
Step 2
Scoped evidence retrieval
The agent loads a bounded issue and policy packet instead of writing from a vague context blob.
Step 3
Gemini tool calling
Internal tools handle the operational facts first. Google Search grounding is only used when fresh external context materially helps.
Step 4
Durable streaming
The UI subscribes to `messageEvents`, so users see plan creation, execution progress, streamed text, sources, and quality checks in the operator rail.

FloorAI / Slide 3

The key design tradeoffs were about workflow fit, not chasing the flashiest stack.

Stack decisions

Decision Why it won Tradeoff
Convex-first backend Realtime issues, files, messages, streaming state, eval artifacts, and audit logs all live together. Dedicated vector systems can be stronger on retrieval depth, but retrieval is not the whole product here.
Harnessed agent, not one-shot chat Bounded evidence + tool calls + trace + quality checks gives much better operational trust. More engineering work than a single LLM call.
Pro-class model selectively Broader regional synthesis and messy multimodal cases benefit from stronger reasoning. Routine cases can often move to cheaper planner or deterministic paths later.
Durable streaming over plain SSE Replay, auditability, and operator-visible progress mattered more than the fastest possible demo setup. More backend and UI complexity.
Convex storage now, R2 later Fast prototype path today, cleaner migration to low-egress object storage when evidence traffic grows. Not the final production storage posture.

What an interviewer can ask next

  • How would you move more traffic to deterministic or Flash-class paths?
  • When would you add LangGraph or Deep Agents for multi-issue orchestration?
  • What changes when real auth replaces demo operator sessions?
  • How would you productionize citations, storage, and eval gating?
